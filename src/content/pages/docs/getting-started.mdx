# Getting Started

Quick guide to using and customizing this boilerplate.

## Domain-aware hreflang

- Update `site` in `astro.config.mjs` to your primary domain.
- Configure per-locale domains in `src/i18n/config.ts` (`domains`), including subdomains, separate ccTLDs, or even external hosts with a base path.
- The boilerplate builds absolute hreflang/canonical URLs from this map (and supports absolute URLs in `alternates` when you need custom targets).
- Astro's built-in `i18n.domains` requires `output: "server"`; this template stays `static` and uses the map for link generation only.

## Installation

```bash
git clone <repo-url> my-site
cd my-site
npm install
npm run dev
```

## Adding Pages

### Auto-Matched (Same Path)

Create files with the same structure:

```
pages/services.mdx      → /services
pages/ru/services.mdx   → /ru/services
```

Linked automatically, no frontmatter needed.

### Custom URLs

When paths must differ:

```yaml
# pages/about.mdx
alternates:
  en: about
  ru: o-nas
```

```yaml
# pages/ru/o-nas.mdx
permalink: o-nas
alternates:
  en: about
  ru: o-nas
```

## Adding a Locale

1. `astro.config.mjs` — add to `locales`
2. `src/i18n/config.ts` — add to `locales` and `localeLabels`
3. `src/i18n/translations.ts` — add UI strings
4. Create `src/content/pages/<locale>/`
