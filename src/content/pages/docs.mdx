## Configuration

Edit `src/data/site.ts`:

```typescript
export const siteUrl = "https://your-domain.com";
export const locales = ["en", "ru"] as const;
export const defaultLocale = "en";

export const template = {
  id: "affiliate",
  style: "paper",
} as const;
```

Edit `src/data/ui.ts` for shared UI strings:

```typescript
export const uiStrings = {
  en: { siteName: "My Site", homeLabel: "Home" },
} as const;
```

## Template System

Templates are self-contained folders under `src/templates/`.

- `Layout.astro` is the global wrapper for all pages.
- `pages/` holds page templates like `Article.astro` and `Product.astro`.
- `components/` contains template-specific UI blocks.
- `styles/` defines style presets for the template.
- `template.ts` declares the template metadata (pages, styles, layout path).

To register a new template:

1. Add a folder under `src/templates/<your-template>/`.
2. Create `template.ts` with `id`, `layout`, `pages`, and `styles`.
3. Add it to `src/templates/registry.ts`.
4. Select it in `src/data/site.ts`.

## Style Presets

Each template can expose multiple style presets. Select one in `src/data/site.ts`:

```typescript
export const template = {
  id: "affiliate",
  style: "studio",
} as const;
```

Add a new style preset by:

1. Adding a new style id in `src/templates/<template>/template.ts`.
2. Creating a dedicated CSS file in `src/templates/<template>/styles/<style>.css`.
3. Defining your variables under `:root[data-template="<template>"][data-style="<style>"]`.
4. (Optional) Import `./base.css` for shared rules.

Each style file is isolated and only the selected one is bundled. Styles can also override or extend base CSS, not just variables.

## Global Components

Global components live in `src/components/global/` and are template-agnostic.

- `head/` is for meta tags, structured data, and other `<head>` markup.
- `body/` is for shared body elements like accessibility helpers.

Templates can import these components and use them without duplicating logic.

## Adding Content

Create MDX files in `src/content/pages/`:

```yaml
---
title: "Page Title"
description: "Meta description for SEO"
---

# Your content here
```

### Frontmatter options

| Field | Type | Description |
|-------|------|-------------|
| title | string | Page title (required) |
| description | string | Meta description |
| updatedAt | date | For sitemap |
| draft | boolean | Exclude from build |
| link | string | External URL (product pages) |
| pros | string[] | Pros list (product pages) |
| cons | string[] | Cons list (product pages) |

## Multilingual Setup

### Same slugs (automatic)

```
pages/about.mdx      -> /about/
pages/ru/about.mdx   -> /ru/about/
```

Hreflang links are generated automatically.

## Template Components

Template-specific components are imported from `src/templates/<template>/components`:

```mdx
import RatingTable from "@/templates/affiliate/components/RatingTable.astro";

<RatingTable collection="software" limit={2} />
```
