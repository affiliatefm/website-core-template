## Структура проекта

```
src/
├── components/global/ # Общие компоненты (head/body)
│   ├── head/
│   └── body/
├── content/pages/     # Ваш контент (MDX-файлы)
│   ├── index.mdx      # Главная страница
│   ├── docs.mdx       # Документация
│   ├── _collections/  # Элементы коллекций (страницы продуктов)
│   ├── software/      # Страница коллекции
│   └── websites/      # Страница коллекции
├── data/              # Ваша конфигурация
│   ├── site.ts        # URL сайта, языки, выбор шаблона
│   └── ui.ts          # Переводы интерфейса
├── templates/         # Шаблоны
│   └── affiliate/     # Пример шаблона
│       ├── layout.astro
│       ├── styles.css
│       ├── i18n.ts
│       ├── pages/
│       │   ├── home.astro
│       │   ├── article.astro
│       │   └── product.astro
│       └── components/
├── i18n/              # Вспомогательные функции для URL
├── integrations/      # Проверки сборки
└── pages/             # Роутер Astro
```

## Конфигурация

Редактируйте `src/data/site.ts`:

```typescript
export const siteUrl = "https://your-domain.com";
export const locales = ["en", "ru"] as const;
export const defaultLocale = "en";

export const template = "affiliate" as const;
```

Редактируйте `src/data/ui.ts` для общих строк интерфейса:

```typescript
export const uiStrings = {
  en: { siteName: "My Site", homeLabel: "Home" },
} as const;
```

## Система шаблонов

Шаблоны — это отдельные папки в `src/templates/`:

- Шаблоны ядра: `src/templates/<id>/` (обновляются через `make update`)
- Пользовательские шаблоны: `src/templates/_<id>/` (сохраняются)

- `layout.astro` — глобальный обёртчик для всех страниц.
- `pages/` содержит шаблоны страниц, такие как `home.astro`, `article.astro` и `product.astro`.
- `components/` — UI-блоки, специфичные для шаблона.
- `styles.css` — опциональный CSS шаблона.
- `i18n.ts` — опциональные строки шаблона.

Идентификатор шаблона — имя папки (используйте нижний регистр). Для пользовательских
шаблонов ведущий символ `_` сохраняется в `src/data/site.ts`.

Чтобы зарегистрировать новый шаблон:

1. Добавьте папку в `src/templates/_<your-template>/`.
2. Добавьте `layout.astro`, `pages/home.astro`, `pages/article.astro`, `pages/product.astro`.
3. Добавьте `styles.css` и/или `i18n.ts`, если нужно.
4. Выберите его в `src/data/site.ts`.

Выбор шаблона страницы:
- Главная (`index.mdx`) → `Home`
- Элементы `_collections` → `Product` (или задайте `collections[].page`)
- Все остальные → `Article`
- Переопределение для страницы: `page` в frontmatter

## Стили

Стили живут внутри шаблона. Добавьте `styles.css` и импортируйте его в `layout.astro`.

## Глобальные компоненты

Глобальные компоненты находятся в `src/components/global/` и не зависят от шаблона.

- `head/` — для мета-тегов, структурированных данных и другого содержимого `<head>`.
- `body/` — для общих элементов body, например, помощников по доступности.

Шаблоны могут импортировать эти компоненты и использовать их без дублирования логики.

## Добавление контента

Создавайте MDX-файлы в `src/content/pages/`:

```yaml
---
title: "Page Title"
description: "Meta description for SEO"
---

# Ваш контент здесь
```

### Опции frontmatter

| Поле | Тип | Описание |
|------|-----|----------|
| title | string | Заголовок страницы (обязателен) |
| description | string | Мета-описание |
| updatedAt | date | Для sitemap |
| draft | boolean | Исключить из сборки |
| link | string | Внешний URL (страницы продуктов) |
| pros | string[] | Список плюсов (страницы продуктов) |
| cons | string[] | Список минусов (страницы продуктов) |
| page | string | Переопределение шаблона страницы ("home", "article", "product") |

## Многоязычная настройка

### Одинаковые слаги (автоматически)

```
pages/about.mdx      -> /about/
pages/ru/about.mdx   -> /ru/about/
```

Hreflang-ссылки генерируются автоматически.

## Компоненты шаблонов

Компоненты шаблонов импортируются из:

- `src/templates/<template>/components` (ядро)
- `src/templates/_<template>/components` (пользовательские)

```mdx
import RatingTable from "@/templates/affiliate/components/RatingTable.astro";

<RatingTable collection="software" limit={2} />
```
